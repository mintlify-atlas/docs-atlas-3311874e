<Info>

  [Bun](https://bun.sh/), or [Deno](https://deno.com/) installed before you
  start.
</Info>

## Create a new Nitro project

The fastest way to get started is the official project scaffolder:

```bash
npx create-nitro-app
```

Follow the prompts to choose a project name and template. Once the CLI finishes, start your development server:

```bash
cd my-app
npm run dev
```

Your server is running at `http://localhost:3000`.

## Add Nitro to an existing Vite project

If you already have a Vite frontend, you can add a full production server without changing your existing setup.

<Steps>
  <Step title="Install dependencies">
    Add `nitro` and `vite` to your project:

    <CodeGroup>

    ```bash npm
    npm install nitro vite
    ```

    ```bash yarn
    yarn add nitro vite
    ```

    ```bash pnpm
    pnpm add nitro vite
    ```

    </CodeGroup>
  </Step>

  <Step title="Add the Nitro plugin to Vite">
    Import and register the Nitro plugin in your `vite.config.ts`:

    ```typescript vite.config.ts
    import { defineConfig } from "vite";
    import { nitro } from "nitro/vite";

    export default defineConfig({
      plugins: [nitro()],
    });
    ```

    This is all Vite needs. Running `vite build` will now produce a `.output/` folder containing both your frontend and backend.
  </Step>

  <Step title="Configure Nitro">
    Create a `nitro.config.ts` at the root of your project to tell Nitro where your server routes live:

    ```typescript nitro.config.ts
    import { defineNitroConfig } from "nitro/config";

    export default defineNitroConfig({
      serverDir: "./server",
    });
    ```

    All route files inside `server/` will be picked up automatically.

    <Tip>
      You can name the server directory anything you like. Update `serverDir`
      to match your project layout.
    </Tip>
  </Step>

  <Step title="Create your first route">
    Add a route handler at `server/api/hello.ts`:

    ```typescript server/api/hello.ts
    import { defineHandler } from "nitro";

    export default defineHandler((event) => {
      return { message: "Hello from Nitro!" };
    });
    ```

    The file path maps directly to the URL — `server/api/hello.ts` becomes `/api/hello`. You can return strings, JSON objects, `Response` instances, or readable streams.
  </Step>

  <Step title="Start the development server">
    <CodeGroup>

    ```bash npm
    npm run dev
    ```

    ```bash yarn
    yarn dev
    ```

    ```bash pnpm
    pnpm dev
    ```

    </CodeGroup>

    Your API route is now available at `http://localhost:3000/api/hello`.

    ```json
    { "message": "Hello from Nitro!" }
    ```
  </Step>
</Steps>

## Standalone Nitro project

If you're building a backend-only server without Vite, use the `nitro` CLI directly.

Add the following scripts to your `package.json`:

```json package.json
{
  "type": "module",
  "scripts": {
    "dev": "nitro dev",
    "build": "nitro build",
    "preview": "node .output/server/index.mjs"
  },
  "devDependencies": {
    "nitro": "latest"
  }
}
```

Then create a `nitro.config.ts`:

```typescript nitro.config.ts
import { defineNitroConfig } from "nitro/config";

export default defineNitroConfig({});
```

Run `npm run dev` to start the development server.

<Note>
  The `nitro dev` command watches your route files and reloads automatically on
  changes — no additional tooling required.
</Note>

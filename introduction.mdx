Nitro gives you a production-ready server with filesystem routing, code-splitting, caching, storage, and a built-in database — all runtime-agnostic and deployable anywhere without changing your code.

## What is Nitro?

Create server and API routes inside a `routes/` directory. Each file maps directly to a URL path, and Nitro handles routing, code-splitting, and optimized builds automatically.

```typescript routes/api/hello.ts
import { defineHandler } from "nitro";

export default defineHandler((event) => {
  return { message: "Hello from Nitro!" };
});
```

The file path maps directly to the route URL — `routes/api/hello.ts` becomes `/api/hello`. No router configuration needed.

<Note>
  You can also take full control of the server entry by creating a `server.ts`
  file and using any HTTP library — [h3](https://h3.dev),
  [Hono](https://hono.dev), [Elysia](https://elysiajs.com/), or plain Web APIs.
</Note>

## Filesystem routing

Routes live in the `routes/` directory. The file path determines the URL:

| File | URL |
|---|---|
| `routes/index.ts` | `/` |
| `routes/api/hello.ts` | `/api/hello` |
| `routes/users/[id].ts` | `/users/:id` |

Nitro compiles these at build time — no runtime router overhead. This makes it especially well-suited for serverless hosting, with near-zero cold-start times regardless of project size.

## Deploy anywhere

`nitro build` produces an optimized `.output/` folder. Point it at any supported runtime:

```bash
# Build for the current platform
nitro build

# Preview the output locally
node .output/server/index.mjs
```

Switch deployment targets by setting the `preset` option in `nitro.config.ts` — no other code changes required. Nitro supports Cloudflare Workers, Vercel Edge Functions, Netlify Functions, AWS Lambda, Bun, Deno, and more.

<Tip>
  See [Deploy](/deploy/index) for the full list of supported providers and
  how to configure each one.

## Vite integration

Nitro works as a Vite plugin. Add it to any existing Vite project to get a full production server alongside your frontend:

```typescript vite.config.ts
import { defineConfig } from "vite";
import { nitro } from "nitro/vite";

export default defineConfig({
  plugins: [nitro()],
});
```

Running `vite build` produces an `.output/` folder containing both your frontend assets and backend server — ready to deploy anywhere as a single unit.

## Built-in storage, caching, and database

Nitro includes a runtime-agnostic key-value storage layer, response caching with stale-while-revalidate support, and a built-in SQL database (SQLite by default). All three use the same driver-swapping model: configure a different driver without changing your application code.

- **Storage** — 20+ drivers including FS, Redis, and S3
- **Cache** — Handler and function caching backed by storage
- **Database** — 10+ SQL databases including Postgres and MySQL

## Meta-framework foundation

Nitro is the server engine behind [Nuxt](https://nuxt.com/), [SolidStart](https://start.solidjs.com/), and [TanStack Start](https://tanstack.com/start). If you're building a meta-framework, Nitro gives you a proven, extensible foundation.
